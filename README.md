# AddFMA — Claude Code Skill

A Claude Code skill for adding new apps and regenerating existing outputs in the [Fleet Maintained Apps (FMA)](https://github.com/fleetdm/fleet/tree/main/ee/maintained-apps) catalog.

## What it does

- **AddApp** — end-to-end workflow for adding a new macOS or Windows app: finds Homebrew/winget metadata, creates input JSON, writes enrichers if needed, runs the ingester, generates an icon, validates the output, commits, and opens a draft PR
- **RegenerateOutput** — reruns the ingester for an existing app to refresh its output JSON
- **MonitorCI** — background agent that polls CI on a draft FMA PR, auto-fixes known failure patterns (up to 2 retries), then marks the PR ready for review when checks pass

## Requirements

- [Claude Code](https://claude.ai/code) (`claude` CLI)
- Go (to run the Fleet ingester)
- `gh` CLI (authenticated to GitHub)
- A local clone of [fleetdm/fleet](https://github.com/fleetdm/fleet)

## Installation

1. Copy the skill into your Claude skills directory:

```bash
git clone https://github.com/tux234/claude-addfma-skill ~/.claude/Skills/AddFMA
```

2. Create your personal preferences file:

```bash
mkdir -p ~/.claude/Skills/PAI/USER/SKILLCUSTOMIZATIONS/AddFMA
cp ~/.claude/Skills/AddFMA/PREFERENCES.md.example \
   ~/.claude/Skills/PAI/USER/SKILLCUSTOMIZATIONS/AddFMA/PREFERENCES.md
```

3. Edit the preferences file and set your values:

```markdown
- `FLEET_REPO` = `/path/to/your/fleet/fleet`
- `GITHUB_USER` = `your-github-username`
```

## Usage

Trigger the skill by describing what you want in natural language:

| What you say | What runs |
|---|---|
| "Add Figma as a Fleet maintained app for macOS" | AddApp workflow |
| "Add Notion for Windows as an FMA" | AddApp workflow |
| "Regenerate the Warp darwin.json" | RegenerateOutput workflow |

Claude reads your PREFERENCES.md and substitutes `{FLEET_REPO}` and `{GITHUB_USER}` throughout the workflow automatically.

## File structure

```
AddFMA/
├── SKILL.md                  # Skill definition and routing
├── PREFERENCES.md.example    # Template for personal config
├── Workflows/
│   ├── AddApp.md             # Add a new app end-to-end
│   └── RegenerateOutput.md   # Regenerate existing output JSON
└── Tools/
    └── MonitorCI.md          # Background CI monitor agent
```

## Notes

- Output JSON files (`darwin.json`, `windows.json`) are always generated by the ingester — never hand-edited
- PRs are opened as drafts and promoted to ready automatically when CI passes
- The MonitorCI agent distinguishes infra flakes (free retries) from real failures (2-retry cap)
